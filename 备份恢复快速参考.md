# 物流报价系统 - 备份恢复快速参考

## 🚀 快速开始

### 执行备份
```bash
# 方法1：界面操作
访问 http://localhost:5173/admin → 备份管理 → 立即备份

# 方法2：API调用
curl -X POST http://localhost:3000/api/backup/execute

# 方法3：脚本执行
node scripts/qiniu-backup.js
```

### 安全恢复 (推荐)
```bash
# 基本用法
node safe-restore.js backup/wlbj-backup-2025-06-14T03-50-15.tar.gz

# 检查服务状态
node check-services.js
```

### 紧急重启
```bash
# 停止服务
pkill -f "npm start"

# 启动服务
npm start

# 验证状态
node check-services.js
```

## 📋 常用命令

### 备份相关
```bash
# 查看备份文件
ls -la backup/wlbj-backup-*.tar.gz

# 验证备份文件
node scripts/verify-backup.js <备份文件>

# 下载最新备份
curl -O http://localhost:3000/api/backup/download
```

### 服务管理
```bash
# 检查服务状态
node check-services.js

# 启动后端服务
npm start

# 查看应用日志
tail -f logs/app.log

# 查看进程
ps aux | grep node
```

### 数据库操作
```bash
# 检查数据库
ls -la data/logistics.db

# 验证数据库完整性
sqlite3 data/logistics.db "PRAGMA integrity_check;"

# 查看数据库表
sqlite3 data/logistics.db ".tables"
```

## 🔧 故障排除

### 服务无法启动
```bash
# 1. 检查端口占用
netstat -tlnp | grep 3000

# 2. 强制停止进程
sudo pkill -9 -f "node"

# 3. 检查数据库权限
chmod 644 data/logistics.db

# 4. 重新启动
npm start
```

### 前端连接错误
```bash
# 1. 检查后端状态
curl http://localhost:3000

# 2. 重启后端
npm start

# 3. 刷新前端页面
```

### 恢复失败
```bash
# 1. 使用恢复前备份回滚
cp backup/pre-restore-*/logistics.db data/logistics.db

# 2. 验证备份文件
node scripts/verify-backup.js <备份文件>

# 3. 重新执行恢复
node safe-restore.js <备份文件>
```
