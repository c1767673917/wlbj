# 物流报价系统异地备份部署指南

## 📋 概述

本指南将帮助您为物流报价系统(wlbj)部署一套完整的异地备份解决方案，确保数据安全和业务连续性。

## 🎯 备份目标

- **RTO (恢复时间目标)**: 2小时内完成数据恢复
- **RPO (恢复点目标)**: 数据丢失不超过1小时
- **可用性**: 99.9%的数据可用性保证
- **安全性**: 端到端加密，多重验证

## 🏗️ 架构设计

### 三层备份架构

```
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│   本地备份      │    │   同城异地      │    │   异地云存储    │
│   (每小时)      │───▶│   (每4小时)     │───▶│   (每天)        │
│   RTO: 5分钟    │    │   RTO: 30分钟   │    │   RTO: 2小时    │
└─────────────────┘    └─────────────────┘    └─────────────────┘
```

## 🚀 快速部署

### 1. 环境准备

```bash
# 创建备份用户
sudo useradd -m -s /bin/bash backup
sudo usermod -aG sudo backup

# 创建备份目录
sudo mkdir -p /backup/wlbj-logistics
sudo chown backup:backup /backup/wlbj-logistics
sudo chmod 750 /backup/wlbj-logistics

# 安装必要工具
sudo apt update
sudo apt install -y sqlite3 curl jq gzip tar
```

### 2. 下载备份脚本

```bash
# 切换到备份用户
sudo su - backup

# 下载备份脚本
cd /home/backup
wget https://raw.githubusercontent.com/your-repo/wlbj/main/backup-system.sh
wget https://raw.githubusercontent.com/your-repo/wlbj/main/restore-system.sh
wget https://raw.githubusercontent.com/your-repo/wlbj/main/backup-monitor.sh
wget https://raw.githubusercontent.com/your-repo/wlbj/main/backup-config.example

# 设置执行权限
chmod +x backup-system.sh restore-system.sh backup-monitor.sh
```

### 3. 配置备份参数

```bash
# 复制配置文件
cp backup-config.example backup-config.conf

# 编辑配置文件
nano backup-config.conf
```

**关键配置项**:
```bash
# 应用路径
APP_DIR="/path/to/your/wlbj"

# 云存储配置 (选择一种)
ALIYUN_OSS_ENABLED=true
ALIYUN_OSS_BUCKET="your-backup-bucket"
ALIYUN_ACCESS_KEY="your-access-key"
ALIYUN_SECRET_KEY="your-secret-key"

# 企业微信通知
WECHAT_WEBHOOK_URL="https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=YOUR_BOT_KEY"
```

### 4. 测试备份功能

```bash
# 测试备份脚本
./backup-system.sh --test

# 手动执行一次备份
./backup-system.sh

# 检查备份结果
ls -la /backup/wlbj-logistics/
```

### 5. 设置定时任务

```bash
# 编辑crontab
crontab -e

# 添加以下任务
# 每小时执行备份
0 * * * * /home/backup/backup-system.sh

# 每天8点执行监控检查
0 8 * * * /home/backup/backup-monitor.sh --cron

# 每周日凌晨2点清理旧备份
0 2 * * 0 find /backup/wlbj-logistics -type d -mtime +7 -exec rm -rf {} \;
```

## 🛠️ 云存储配置

### 阿里云OSS配置

1. **创建OSS Bucket**:
   ```bash
   # 安装ossutil
   wget https://gosspublic.alicdn.com/ossutil/1.7.14/ossutil64
   chmod +x ossutil64
   sudo mv ossutil64 /usr/local/bin/ossutil
   
   # 配置ossutil
   ossutil config
   ```

2. **设置生命周期规则**:
   - 标准存储: 30天
   - 低频访问: 90天
   - 归档存储: 永久

### 腾讯云COS配置

1. **安装coscli工具**:
   ```bash
   wget https://github.com/tencentyun/coscli/releases/download/v0.13.0-beta/coscli-linux
   chmod +x coscli-linux
   sudo mv coscli-linux /usr/local/bin/coscli
   ```

2. **配置访问凭证**:
   ```bash
   coscli config set -s your-secret-id -k your-secret-key -r ap-guangzhou
   ```

### 七牛云配置

1. **安装qshell工具**:
   ```bash
   wget https://devtools.qiniu.com/qshell-v2.11.0-linux-amd64.tar.gz
   tar -xzf qshell-v2.11.0-linux-amd64.tar.gz
   sudo mv qshell /usr/local/bin/
   ```

2. **配置账号**:
   ```bash
   qshell account your-access-key your-secret-key backup-account
   ```

## 🔧 运维操作

### 日常检查

```bash
# 检查备份状态
./backup-monitor.sh

# 查看备份列表
./restore-system.sh --list

# 检查磁盘空间
df -h /backup

# 查看备份日志
tail -f /var/log/backup-wlbj-logistics.log
```

### 数据恢复

1. **查看可用备份**:
   ```bash
   ./restore-system.sh --list
   ```

2. **恢复完整数据**:
   ```bash
   ./restore-system.sh 20241201_143000
   ```

3. **仅恢复数据库**:
   ```bash
   ./restore-system.sh -d 20241201_143000
   ```

4. **从云端恢复**:
   ```bash
   # 先从云端下载备份
   ossutil cp -r oss://your-bucket/backups/wlbj-logistics/20241201_143000 /backup/wlbj-logistics/
   
   # 然后执行恢复
   ./restore-system.sh 20241201_143000
   ```

### 故障处理

1. **备份失败**:
   ```bash
   # 检查日志
   tail -100 /var/log/backup-wlbj-logistics.log
   
   # 检查磁盘空间
   df -h
   
   # 检查权限
   ls -la /backup/wlbj-logistics/
   ```

2. **云端上传失败**:
   ```bash
   # 检查网络连接
   ping oss-cn-hangzhou.aliyuncs.com
   
   # 检查凭证配置
   ossutil ls oss://your-bucket/
   
   # 手动重试上传
   ./backup-system.sh --retry-upload
   ```

## 📈 性能优化

### 备份优化

1. **压缩优化**:
   ```bash
   # 在backup-config.conf中设置
   COMPRESSION_LEVEL=6
   COMPRESSION_THREADS=2
   ```


## 🔒 安全加固

### 数据加密

1. **传输加密**:
   - 所有云存储均使用HTTPS/TLS
   - 启用SSL证书验证

2. **存储加密**:
   ```bash
   # 启用本地加密
   ENCRYPTION_ENABLED=true
   ENCRYPTION_KEY="your-32-char-encryption-key"
   ```

### 访问控制

1. **最小权限原则**:
   ```bash
   # 备份用户只有必要权限
   sudo usermod -G backup backup
   ```

2. **密钥管理**:
   ```bash
   # 使用环境变量存储敏感信息
   export ALIYUN_ACCESS_KEY="your-key"
   export ALIYUN_SECRET_KEY="your-secret"
   ```
