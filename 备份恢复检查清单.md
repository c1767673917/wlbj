## 🔄 恢复操作检查清单

### 恢复前准备 ✅
- [ ] 确认恢复需求和范围
- [ ] 选择正确的备份文件
- [ ] 验证备份文件完整性 (`node scripts/verify-backup.js`)
- [ ] 确认当前系统状态
- [ ] 通知相关人员维护窗口

### 安全恢复执行 ✅
- [ ] 进入项目目录 (`cd /path/to/wlbj`)
- [ ] 检查当前服务状态 (`node check-services.js`)
- [ ] 执行安全恢复脚本 (`node safe-restore.js <备份文件>`)
- [ ] 监控恢复进度和日志输出
- [ ] 等待恢复完成

### 恢复过程监控 ✅

- [ ] 验证恢复前备份创建
- [ ] 监控数据恢复进度
- [ ] 确认服务自动重启
- [ ] 检查API响应正常

### 恢复后验证 ✅
- [ ] 检查服务状态 (`node check-services.js`)
- [ ] 验证前端访问 (`http://localhost:5173`)
- [ ] 验证后端API (`http://localhost:3000`)
- [ ] 测试管理员登录 (`http://localhost:5173/admin`)
- [ ] 检查数据库完整性 (`sqlite3 data/logistics.db "PRAGMA integrity_check;"`)

### 功能验证 ✅

- [ ] 通知相关人员恢复完成

## 🔍 故障排除检查清单

### 服务启动失败 ✅
- [ ] 检查端口占用 (`netstat -tlnp | grep 3000`)

- [ ] 强制重启服务
