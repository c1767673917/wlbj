## 🔧 问题说明

### 为什么需要安全恢复？

1. **服务冲突**：在线恢复时，后端服务仍在运行，可能导致数据库锁定冲突
2. **进程崩溃**：直接覆盖数据库文件会导致正在运行的服务异常退出
3. **数据不一致**：服务运行时恢复可能导致数据状态不一致

### 常见错误现象

```
[vite] http proxy error: /api/backup/config
AggregateError [ECONNREFUSED]:
    at internalConnectMultiple (node:net:1139:18)
    at afterConnectMultiple (node:net:1714:7)
```

这表示后端服务已经停止，通常是恢复操作导致的。

## ✅ 安全恢复脚本 (推荐方法)

### 1. 脚本功能特性

`safe-restore.js` 脚本提供以下功能：

- ✅ **智能进程检测**：自动检测正在运行的Node.js进程
- ✅ **优雅停止服务**：安全停止相关服务，避免数据损坏
- ✅ **创建恢复前备份**：自动备份当前数据，支持回滚
- ✅ **安全执行恢复**：在服务停止状态下恢复数据
- ✅ **自动重启服务**：恢复完成后重新启动服务
- ✅ **验证恢复结果**：检查服务状态和API响应
- ✅ **详细进度显示**：实时显示恢复过程和状态

### 2. 使用步骤详解

#### 步骤1：准备工作
```bash

# 基本用法
```

## 🔄 服务恢复方法

如果服务因为恢复操作而停止，可以使用以下方法：

### 方法1：使用检查脚本 (推荐)
```bash
cd /path/to/wlbj
node check-services.js
```

### 方法2：手动重启服务
```bash
cd /path/to/wlbj
npm start
```

### 方法3：完整重启流程
```bash
# 1. 停止所有相关进程
pkill -f "npm start"
pkill -f "node.*server"

# 2. 等待进程完全停止
sleep 3

# 3. 重新启动后端服务
npm start

# 4. 验证服务状态
node check-services.js
```

### 方法4：强制重启 (最后手段)
```bash
# 强制杀死所有Node.js进程
sudo pkill -9 -f "node"

# 等待
sleep 5

# 重新启动
npm start
```

## 🛠️ 故障排除

### 常见问题及解决方案

#### 问题1：安全恢复脚本无法停止服务
**现象**：脚本提示无法停止正在运行的服务

**解决方案**：
```bash
# 方法1：手动强制停止
sudo pkill -9 -f "npm start"
sudo pkill -9 -f "node.*server"

# 等待进程完全停止
sleep 3

# 然后重新运行安全恢复
node safe-restore.js <备份文件>
```

```bash
# 方法2：检查具体进程
ps aux | grep node
# 记录PID，然后逐个杀死
kill -9 <PID>
```

#### 问题2：恢复后服务无法启动
**现象**：恢复完成后，`npm start` 失败

**解决方案**：
```bash
# 1. 检查数据库文件
ls -la data/logistics.db
# 确保文件存在且大小正常

# 2. 检查文件权限
chmod 644 data/logistics.db
chown $USER:$USER data/logistics.db

# 3. 检查数据库完整性
sqlite3 data/logistics.db "PRAGMA integrity_check;"

# 4. 手动启动并查看详细错误
npm start
```

#### 问题3：恢复后数据异常或丢失
**现象**：恢复后登录失败或数据不完整

**解决方案**：
```bash
# 1. 使用恢复前备份回滚
cp backup/pre-restore-*/logistics.db data/logistics.db

# 2. 恢复配置文件
cp backup/pre-restore-*/.env ./
cp backup/pre-restore-*/auth_config.json ./

# 3. 重新启动服务
npm start

# 4. 如果问题依然存在，检查备份文件完整性
node scripts/verify-backup.js <备份文件>
```

#### 问题4：前端连接错误
**现象**：前端显示大量代理错误

**解决方案**：
```bash
# 1. 检查后端服务状态
node check-services.js

# 2. 如果后端未运行，重启服务
npm start

# 3. 等待服务完全启动后刷新前端页面
```

#### 问题5：备份文件损坏或无效
**现象**：恢复时提示备份文件无效

**解决方案**：
```bash
# 1. 验证备份文件
node scripts/verify-backup.js <备份文件>

# 2. 检查文件完整性
tar -tzf <备份文件> | head -10

# 3. 重新下载备份文件
curl -O http://localhost:3000/api/backup/download

# 4. 使用其他备份文件
ls -la backup/wlbj-backup-*.tar.gz
```
